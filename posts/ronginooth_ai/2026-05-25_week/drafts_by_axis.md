# 5軸 × 5本 ドラフト集（2026-05-25_week 用）

> 各軸 5本、合計25本。フック型と派生を変え、誰に向けて書くかを明記。
> ここから14本（または任意数）を最終キューに選抜する。

---

## 📖 論文軸（教授エディター視点）

### 論文-1: 教授エディター reveal
**フック**: 対話型・舞台裏 reveal
**誰に向けて**: 論文を「出す側」しか経験がない若手研究者・大学院生

```
教授がエディターをやっているのは、知らなかった。

土曜日、研究室にこもっている理由を聞いた。

「論文を読んで、査読者を探して、メールする」

先週は、依頼した相手から「共著者なのでできません」と断られたらしい。

——これで無給ですか？

「そう、無給」

論文を出す側にいると、見えない裏側がある。
```

---

### 論文-2: エディター目線で書く
**フック**: 気づき型・リスト型
**誰に向けて**: 投稿しても editor desk reject される研究者・査読まで届かない人

```
査読と編集の苦労を見たら、論文の書き方が変わった。

エディターが読むのは、4つだけ：

カバーレター
タイトル
アブストラクト
図（流し読み）

ここで「専門家が誰か」が伝わらないと、査読に回らない。

書く相手が変われば、書き方も変わる。
```

---

### 論文-3: 査読を断った先で誰かが土曜
**フック**: 二択型・engagement 狙い
**誰に向けて**: 査読を頼まれる中堅研究者・断り方に迷っている人

```
査読を依頼されて、断ったこと、ありますか？

①ある（時間がない）
②ある（共著者・利益相反）
③ない（来ない）

エディター側は、断られるたびに次の候補を探している。
査読を断った先で、誰かが土曜日を使っている。
```

---

### 論文-4: 誰のための Introduction
**フック**: 失敗告白型・対話型
**誰に向けて**: Introduction で詰まり続けている博士課程・若手

```
指導教員に「この Introduction、誰のための論文？」と聞かれた。

言葉に詰まった。

自分のための Introduction を、書いていた。

読者は2段落で離脱する——その前提で書き直したら、論文が変わった。

あなたは、誰のために書いていますか？
```

---

### 論文-5: AI で書ける時代の本質
**フック**: open question 型
**誰に向けて**: AIで論文を書き始めたが「本当にこれでいいのか」と不安な研究者

```
AI で論文が書けるようになって、「書けない」という感覚はほとんど消えた。

でも、書く前に決まっていることがある。

「誰に読ませるか」「何を専門家に伝えるか」

ここを設計しないまま AI に書かせると、長くて流し読みされる論文が出てくる。

あなたは、書く前に何を決めていますか？
```

## 📊 図表軸（R × AI → R-note 導線）

### 図表-1: EZR の jpeg 査読指摘
**フック**: 対話型・失敗談
**誰に向けて**: EZR・Excel で論文図を作っていて修正指示が来た研究者

```
査読者に言われた言葉。

「Figure 1 は R で出していますか？」

出していなかった。EZR だった。

答えられなかった、初めての査読コメントだった。

あの日から、「ツールの問題」を疑うようになった。
```

---

### 図表-2: 後輩が CSV から図（R-note 導線）
**フック**: 後輩エピソード・L3 導線
**誰に向けて**: R を諦めて、後輩や指導学生に背中を見せられずにいる研究者

```
「先生、CSV から3群比較の図が出ました」

満面の笑みで来た後輩。

私が渡したのは、プロンプトではない。
R の環境を設定する、ある1枚のファイルだった。

詳しくは、プロフのリンクから。
```

---

### 図表-3: 解像度地獄
**フック**: あるある型・問いかけ
**誰に向けて**: 投稿のたびに「tiff で」「解像度を上げて」と言われる研究者

```
論文図の投稿前チェックで、毎回同じ指示が来る。

「解像度を上げて」「フォントを大きく」「tiff で提出を」

EZR の jpeg では出せない、と気づいたのは、3本目の論文でだった。

あなたは何で出していますか？
```

---

### 図表-4: 設計図1枚の威力
**フック**: 体験談型・数字フック・教育
**誰に向けて**: AI に CSV を渡したが「思った図」と違うものが出てきた経験のある人

```
発表2年前まで、3群比較の図を作るのに丸1日かかっていた。

今は30分で出る。変えたのは1つだけ。

CSV を AI に渡す前に、「出力形式の設計図」を1枚作るようにした。

R のコードを書いていない。設計図を渡しただけ。
```

---

### 図表-5: 勉強より先に1枚出す（L3 強導線）
**フック**: 逆張り型・L3 導線
**誰に向けて**: 「R を勉強してから使おう」と3年止まっている人

```
「R を勉強してから使おう」が危険な理由がある。

環境構築・エラー・tidy data・ggplot2 の文法が、最初から一気に来る。

本業を抱えながら越えるには、コストが高すぎる。

越え方は1つ。勉強より先に、1枚図を出すこと。

その方法、プロフにまとめています。
```

## 📝 申請書軸（量で勝つ）

### 申請書-1: 気合い→量
**フック**: 対話型・逆説
**誰に向けて**: 科研費に何回も落ちて気合いを入れて書いている研究者

```
研究費を取り続けている人に聞いた言葉。

「気合い入れて書いたのほど、落ちてた」

——では何が変わったんですか？

「とにかく出している。それだけ」

完璧に書いてから出す、をやめた日から、採択率が変わるらしい。

あなたは去年、何本出しましたか？
```

---

### 申請書-2: 書きすぎて止まる
**フック**: あるある型・気づき
**誰に向けて**: 推敲しすぎて結局申請しなかった年がある研究者

```
申請書、書きすぎて止まっていた。

「もっと推敲してから」「もう少し論理を詰めてから」

——結局、出さなかった年もあった。

通っている人は、書く量ではなく、出す回数で稼いでいた。
```

---

### 申請書-3: タイトルだけ変えて再提出
**フック**: 失敗→復活型・短文パンチ
**誰に向けて**: 1回落ちて諦めて翌年出さなかった人

```
科研費、1回落ちた。

翌年、タイトルだけ変えて出し直した。

採択された。

（正確には中身も少しブラッシュアップした）

「落ちた」を「修正版」に変えるだけで、再起のハードルが下がる。
```

---

### 申請書-4: 評価一致率34%
**フック**: びっくり数字＋根拠
**誰に向けて**: 不採択の理由を「内容が悪かった」と思い込んでいる研究者

```
科研費の審査員間の評価一致率は34%。
（Pier et al., 2018, PNAS）

つまり、採択・不採択の境界の半分は、運。

「落ちたから内容が悪い」ではない。
「次も出す」が、最後の戦略になる。
```

---

### 申請書-5: 流し読みで分かる構成
**フック**: 二択型・教育
**誰に向けて**: 申請書を「丁寧に書く＝伝わる」と思っている人

```
採択される申請書、どちらだと思いますか？

①論理を詰めて、隙のない記述
②流し読みで全体が分かる構成

——通っている人に聞いたら、答えは②だった。

審査員も、時間のない研究者だった。
```

## 📈 データ解析軸（PCA 2週間 → 5分）

### データ-1: PCA 革命
**フック**: 体験談型・数字フック（2週間 → 5分）
**誰に向けて**: PCA や多変量解析で「R を使えればいいのに」と止まっている研究者

```
10年前、論文投稿で必要になった PCA 解析。

ネットの情報を探り、コードを直し、エラーを潰し——2週間かかった。

今、同じ解析を AI に頼んでいる。

指示30秒。完成まで、修正を入れて5分。

革命だ。
```

---

### データ-2: R-note 導線
**フック**: L3 導線型・過去の自分に向けて
**誰に向けて**: R を諦めて EZR・Excel に戻った研究者

```
10年前、PCA 解析で2週間溶かした。

ネットの情報を集め、コードを直し、エラーで諦めかけた。

同じ解析を、今は AI が5分で出す。

「R を勉強する時間がない」と思っている人に、たどり着いた方法をまとめました。

プロフのリンクから。
```

---

### データ-3: 検定の前に図を見たか
**フック**: 対話型・教育
**誰に向けて**: p 値から解析に入ってしまう若手・大学院生

```
統計の先生に言われた言葉。

「検定の前に、図を見ましたか？」

見ていなかった。p 値から入っていた。

「データに聞いてから、数字で確かめる」

その日から、解析の順番が変わった。
```

---

### データ-4: 解析ツール 二択
**フック**: 二択型・survey 型（過去3リプ実績あり型の派生）
**誰に向けて**: 解析ツール選びに迷っている全研究者

```
研究者の皆さんに聞きたい。

論文のデータ解析、何でやっていますか？

① Excel
② EZR / SPSS
③ R / Python（自力）
④ R / Python（AI 活用）

私は ①→②→④。変えたのはツールではなく、渡し方だった。
```

---

### データ-5: 検定をコピペで通過していた頃
**フック**: 失敗談型・査読指摘
**誰に向けて**: 投稿論文のレビューで「統計に問題」と言われた経験のある人

```
査読コメント。

「統計手法が不適切。Mann-Whitney ではなく、Kruskal-Wallis を使うべき」

その時、なぜ Mann-Whitney を選んだかすら、説明できなかった。

統計を「コピペで通過するもの」と思っていた頃の話。

今は AI に「このデザインで適切な検定は？」と聞いてから入る。
```

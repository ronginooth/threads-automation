# Threads自動投稿停止の原因と修正計画

## 調査結果
- **原因**: 3月26日のリファクタリング時に `.gitignore` に以下の行が追加され、生成した投稿文がGitの追跡対象から外れていました。
  ```gitignore
  data/*/queue/*.md
  data/*/posted/*.md
  ```
- **詳細**: ローカルでは投稿文（4月上旬〜今週分）が正常に生成され `data/ronginooth_ai/queue/` に保存されていますが、上記の `.gitignore` の影響により `git push` してもGitHubに送信されていませんでした。その結果、サーバー上のキューが空になり、GitHub Actionsからの投稿がストップしていました。

## User Review Required

> [!WARNING]
> `.gitignore` を修正してプッシュすると、GitHubアクションが再開します。現在のローカルのキューには「4月5日」など過去に設定されたスケジュール日時のファイルが大量に残っています。
> `post.py` の仕様ではスケジュールの時間を過ぎたものが投稿されるため、プッシュ直後から順次（または一日に規定された上限の件数まで）**自動的に過去の投稿が連発される可能性があります**。
> 実行前に、**ローカルに溜まっているキューファイルの `scheduled` の日時を今日以降の日付に書き換える作業（再スケジュール）を実施しますか？** そのままプッシュしてもよいか、ご指示をお願いします。

## Proposed Changes

### 1. `.gitignore`

#### [MODIFY] .gitignore (file:///Users/makino/local-vaults/claude-code-starter-kit/threads-automation/.gitignore)
`data/*/queue/*.md` および `data/*/posted/*.md` を削除（またはコメントアウト）し、GitHub Actionsがファイルを参照できるように追跡を許可します。

### 2. リモートと同期の実行 (Gitコマンド)
現在、ローカルがGitHub上より約450コミット遅れている（Actionsによる自動更新）ため、以下の順で安全に同期およびプッシュを行います。
1. `git pull --rebase origin main`（最新を取り込む）
2. `git add .gitignore data/ronginooth_ai/queue/ data/ronginooth_ai/posted/`
3. `git commit -m "fix(automation): unignore queue files and push cached posts"`
4. `git push origin main`
